# 塔羅牌解讀 — 代碼生成範本（給協作 AI 用）

請按以下格式輸出**單張牌**的 TypeScript 物件，可直接貼入 `src/lib/tarotCards.ts` 的對應陣列中。每次只生成一張牌，不要改欄位名稱或結構。

---

## 一、欄位說明

| 欄位 | 類型 | 必填 | 說明 |
|------|------|------|------|
| `id` | string | ✓ | 大牌：`major_0`～`major_21`（0=愚者…21=世界）；小牌見下方 id 對照 |
| `nameZh` | string | ✓ | 中文牌名，如 `'愚者'`、`'權杖五'` |
| `suit` | string | ✓ | 大牌固定 `'major'`；小牌為 `'wands'` / `'cups'` / `'swords'` / `'pentacles'` |
| `upright` | string | ✓ | 正位牌義，一整段（可含牌面故事 + 心理/情境） |
| `reversed` | string | ✓ | 逆位牌義，一整段 |
| `keywords` | string[] | ✓ | 約 5 個關鍵詞，如 `['豐饒', '母性', '感官享受', '創造力', '自然節律']` |
| `advice` | string | ✓ | **整體**一句建議（總收束） |
| `love` | string | ✓ | 愛情運勢：正逆位解讀 + 段末**建議：……**（該領域具體建議） |
| `career` | string | ✓ | 事業運勢：正逆位解讀 + 段末**建議：……** |
| `wealth` | string | ✓ | 財運：正逆位解讀 + 段末**建議：……** |

- 所有內容使用**繁體中文**。
- `love` / `career` / `wealth` **必須是單一字串**（不要用 `{ description, advice }` 物件），建議寫在該段文字結尾，例如：「……逆位時則要警惕……。建議：用你的肩膀扛起責任……」
- 字串內若有單引號，請用 `\'` 或改寫，避免斷句。

---

## 二、大牌 id 對照（0～21）

0 愚者、1 魔術師、2 女祭司、3 皇后、4 皇帝、5 教皇、6 戀人、7 戰車、8 力量、9 隱者、10 命運之輪、11 正義、12 倒吊人、13 死神、14 節制、15 惡魔、16 高塔、17 星星、18 月亮、19 太陽、20 審判、21 世界。

---

## 三、小牌 id 格式（若之後要生成小牌）

- 權杖：`wands_01`～`wands_14`（01–10 為數字牌，11–14 為侍從/騎士/皇后/國王）
- 聖杯：`cups_01`～`cups_14`
- 寶劍：`swords_01`～`swords_14`
- 錢幣：`pentacles_01`～`pentacles_14`

## 五、輸出要求

- 只輸出**一個物件**，從 `{` 到 `},`（結尾保留逗號，方便貼入陣列）。
- 不要包含 `export`、`const` 或外層陣列括號。
- 牌義風格：專業、有畫面感、正逆位與愛情/事業/財運都要寫滿，且每塊結尾有「建議：……」。

把本文件與上面範例一併提供給生成塔羅解讀的 AI，請其按此格式直接輸出代碼即可。
