# 客戶端閃退排查

## 1. 是哪裡閃退？

| 情況 | 可能原因 | 處理方式 |
|------|----------|----------|
| **Cursor 編輯器** 自己關掉 | 記憶體不足、擴展過多 | 見下方「Cursor 閃退」 |
| **終端裡 `npm run dev` 崩掉** | Node 記憶體不足 | 用 `npm run dev:stable` 或加大 Node 記憶體 |
| **瀏覽器分頁** 開專案時崩掉 / 白屏 | 分頁記憶體爆、或某頁面死循環 | 見下方「瀏覽器閃退」 |

---

## 2. Cursor 編輯器閃退（含 OOM）

若彈窗顯示 **「窗口意外終止」、原因: `oom`、代碼: `-536870904`**，就是 **記憶體不足 (Out Of Memory)**，Cursor 進程吃滿記憶體被系統終止。

### 立刻可做的

1. **關掉不用的編輯器分頁**  
   只保留當前在改的 1～2 個檔案，其餘關閉（尤其大檔案、JSON、日誌）。

2. **關閉或停用擴展**  
   `Ctrl+Shift+X` → 停用暫時用不到的擴展（如多餘的主題、語言包、大型擴展），重載視窗。

3. **少開其他吃記憶體的程式**  
   關掉多餘的 Chrome 分頁、其他 IDE、遊戲等，讓系統多留記憶體給 Cursor。

4. **用「重新打開」恢復**  
   崩潰後點「重新打開(R)」即可從上次位置繼續；若勾選「不還原編輯器」再開，會更省一點記憶體。

### 長期減輕負擔

- **不要用「打開資料夾」開整個磁碟或超大目錄**，只開專案目錄（如 `D:\Dmy-projects`）。
- **排除不需索引的資料夾**：在專案根目錄加 `.cursorignore` 或確保 `node_modules`、`dist`、`.git` 等被排除，減少 Cursor 索引量。
- **本機記憶體 ≤ 8GB**：考慮加裝記憶體，或改用較輕量的編輯器做部分工作。

## 3. 開發伺服器（Vite）閃退

專案已提供較省記憶體的啟動方式：

```bash
npm run dev:stable
```

或手動加大 Node 記憶體後再跑：

```bash
# Windows PowerShell
$env:NODE_OPTIONS="--max-old-space-size=4096"; npm run dev

# Windows CMD
set NODE_OPTIONS=--max-old-space-size=4096 && npm run dev
```

可把 `start-dev.bat` 裡的 `npm run dev` 改成 `npm run dev:stable`，用批次檔啟動時也會比較穩。

## 4. 瀏覽器分頁閃退 / 白屏

- **只開一個專案分頁**，關掉重複的 localhost 分頁。
- **硬體加速**：Chrome → 設定 → 系統 → 關閉「使用硬體加速」試試（有時顯卡驅動會導致崩潰）。
- **無痕模式** 開 http://localhost:8081 試一次，排除擴展干擾。
- 若**只有某個頁面**（例如塔羅、關係頁）必崩：可能是該頁動畫或資料量過大，可回報哪一頁，再針對性優化。
