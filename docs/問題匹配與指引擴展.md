# 問題匹配與指引擴展說明

目標：**讓用戶的具體問題能對應到貼題的答案**，且新增問法時**只改配置、不改解讀邏輯**。

---

## 一、當前流程（兩層）

1. **大類匹配**（`questionCategoryKeywords.ts`）  
   用戶問題 → 是否包含某分類的任一關鍵詞 → 得到一個大類（如 身体健康、感情婚姻、事业工作）。

2. **主題指引**（`questionGuidanceTopics.ts`）  
   在大類下再按「主題」細分：同一大類內多條規則（pattern + guidance），問題命中哪條就用哪條的指引；都不命中則用該大類的預設一句。

解讀組件（雷諾曼、塔羅等）只調用 `detectQuestionCategory(question)` 和 `getCategoryGuidance(question, category)`，不關心關鍵詞和主題表長什麼樣。

---

## 二、如何新增「具體問題 → 對應答案」

### 情況 A：用戶問法屬於已有大類，但落進「其他」

- **原因**：大類關鍵詞沒覆蓋到這種說法。  
- **做法**：在 `questionCategoryKeywords.ts` 裡，找到對應大類的 `keywords` 數組，**追加幾個關鍵詞**（或同義說法）。  
- **無需**：改 `questionClassification.ts` 或任何解讀組件。

### 情況 B：大類對了，但希望這種「具體問法」有專用指引

- **原因**：大類預設只有一句，太籠統；你想讓「寵物生病」「加薪」「複合」等有各自一句。  
- **做法**：在 `questionGuidanceTopics.ts` 裡，找到對應大類（如 `身体健康`、`感情婚姻`），在該分類的 `topics` 數組裡**追加一筆**：
  - `pattern`：正則 `RegExp` 或關鍵詞數組 `string[]`（問題包含任一即命中）。
  - `guidance`：這類問法專用的一句指引。
- **無需**：改 `questionClassification.ts` 或解讀組件。

### 情況 C：想新增一整個大類

- **做法**：  
  1. 在 `questionCategoryKeywords.ts` 的 `QuestionCategory` 和 `CATEGORY_KEYWORDS` 裡新增該大類與關鍵詞。  
  2. 在 `questionClassification.ts` 的 `CATEGORY_LABELS` 和 `CATEGORY_GUIDANCE` 裡新增該大類的標籤與預設指引。  
  3. 若該大類下也要分主題，在 `questionGuidanceTopics.ts` 的 `GUIDANCE_TOPICS` 裡為該大類加一組 `TopicRule[]`。

## 三、主題表已覆蓋的範圍（可照此格式繼續擴）

| 大類       | 主題示例                     |
|------------|------------------------------|
| 身体健康   | 寵物、就醫/手術、慢性/調理   |
| 感情婚姻   | 複合、出軌、暗戀、婚姻危機   |
| 事业工作   | 加薪晉升、跳槽求職、創業項目 |
| 出行迁居   | 車況/故障、搬家置產          |
| 学业考试   | 升學考試、子女學業           |
| 财运投资   | 投資理財、債務要債           |
| 生育子女   | 懷孕備孕、子女健康與教養     |

往後只要在對應大類的 `topics` 裡按需追加 `{ pattern, guidance }` 即可，無需再為「某一句用戶問法」改解讀邏輯。
