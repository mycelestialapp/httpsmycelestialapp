# 如何確認「訂閱可見」是否生效

星盤/行星/宮位的「訂閱可見」依賴資料庫欄位 `profiles.oracle_subscriber`。訂閱用戶該欄位為 `true` 時，會直接看到完整解讀，無需再點解鎖。

---

## 一、確認欄位是否存在

1. 打開 [Supabase Dashboard](https://supabase.com/dashboard)，選中你的專案。
2. 左側點 **Table Editor** → 選 **profiles** 表。
3. 看表頭是否有 **oracle_subscriber** 這一列。
   - **沒有**：需要跑一次遷移（見下方「跑遷移」）。
   - **有**：直接做第二步。

### 跑遷移（若欄位不存在）

在專案根目錄執行：

```bash
npx supabase db push
```

或在本機用 Supabase CLI 跑：

```bash
supabase migration up
```

遷移檔在：`supabase/migrations/20260310000000_add_oracle_subscriber.sql`，會為 `profiles` 增加 `oracle_subscriber`（預設 `false`）。

---

## 二、在後台看某個用戶是否訂閱

1. **Table Editor** → **profiles**。
2. 找到要查的那個用戶（可按 `id` 或 `display_name` 搜尋）。
3. 看該行的 **oracle_subscriber**：
   - `true` → 該用戶會看到完整星盤/宮位/行星解讀。
   - `false` 或空 → 只會看到「訂閱可見」、內容鎖住。

---

## 三、手動把某個用戶設為訂閱（測試用）

### 方式 A：在 Table Editor 裡改

1. **Table Editor** → **profiles** → 找到該用戶那一行。
2. 點 **oracle_subscriber** 格子，改成 **true**，存檔。

### 方式 B：用 SQL 改

1. 左側點 **SQL Editor**，新增查詢。
2. 執行（把 `你的用戶UUID` 換成真實的 `profiles.id`，通常是 `auth.users.id`）：

```sql
UPDATE public.profiles
SET oracle_subscriber = true
WHERE id = '你的用戶UUID';
```

3. 該用戶重新整理占星/星盤頁，應會直接看到完整解讀。

### 如何拿到「用戶 UUID」

- 在 **Authentication** → **Users** 裡點進該用戶，複製 **User UID**；或  
- 在 **Table Editor** → **profiles** 裡，該用戶的 **id** 就是 UUID。

---

## 四、正式上線：誰來把 oracle_subscriber 設成 true？

目前專案裡**沒有**在付費成功時自動寫入 `oracle_subscriber`，需要你接好「訂閱成功」的流程，例如：

- **Stripe**：在 Stripe 的 Webhook（例如 `checkout.session.completed`）裡，收到付款成功後，用 Supabase Admin API 或 Edge Function 對該用戶執行  
  `UPDATE profiles SET oracle_subscriber = true WHERE id = ?`
- **手動/後台**：在自建後台或定時任務裡，根據訂閱名單更新對應的 `profiles.oracle_subscriber`。

接好後，訂閱用戶登入就會直接看到完整內容，無需再點「解鎖」。
