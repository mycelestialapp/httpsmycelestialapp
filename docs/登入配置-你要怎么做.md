# 登入配置：你要怎么做（详细步骤）

按下面顺序做，做完一项打勾一项，全部做完后 Google / Apple / Facebook / X / 邮箱 才能稳定登录。

---

## 第一步：准备好 Supabase 和回調地址

1. 打开浏览器，打开：**https://supabase.com/dashboard**
2. 登录后，选中你的项目（My Celestial 用的那个项目）。
3. 记下这个地址（后面会反复用到，建议复制保存）：

   ```
   https://qnlciqqimwzlsvmplskq.supabase.co/auth/v1/callback
   ```

   在 **Google / Apple / Facebook / X** 的后台里，要填的「回調 / Redirect URI」都是上面这一行，**不要**填 `https://mycelestial.app/...`。

---

## 第二步：Supabase 里配置 URL（必做，否则 OAuth 一定登不上）

1. 在 Supabase 项目里，左侧点 **Authentication**。
2. 再点 **URL Configuration**（或「网址配置」）。
3. **Site URL** 这一栏改成：
   ```
   https://mycelestial.app
   ```
4. 在 **Redirect URLs** 里点「Add URL」，依次添加下面三行（每行添加一次）：
   - `https://mycelestial.app/**`
   - `http://localhost:8080/**`
   - `http://localhost:5173/**`
5. 点 **Save** 保存。

不做这一步，用户用 Google/Apple 等登录后会跳转失败或报错。

---

## 第三步：环境变量（本地和线上都要有）

前端要能连上你的 Supabase，否则任何登录都会失败。

1. 在项目根目录（和 `package.json` 同级）找到或新建 **`.env`** 或 **`.env.local`**。
2. 打开 Supabase：**Settings → API**，复制：
   - **Project URL**（例如 `https://qnlciqqimwzlsvmplskq.supabase.co`）
   - **anon public** 这个 key（一长串字符）
3. 在 `.env` 里写（把下面的值换成你复制的）：

   ```
   VITE_SUPABASE_URL=https://qnlciqqimwzlsvmplskq.supabase.co
   VITE_SUPABASE_PUBLISHABLE_KEY=你的anon_key长串
   ```

4. 如果网站部署在 Vercel / Netlify 等，在托管平台的「环境变量」里同样添加上面两个变量名和值。
5. `.env` 不要提交到 Git（一般已在 `.gitignore` 里）。

---

## 第四步：配置 Google 登录（要用 Google 才做）

### 4.1 在 Google 后台

1. 打开 **https://console.cloud.google.com/**
2. 选一个项目（或新建）→ 左侧 **APIs & Services** → **Credentials**。
3. 点 **Create Credentials** → **OAuth client ID**。
4. 应用类型选 **Web application**。
5. 在 **Authorized redirect URIs** 里点「Add URI」，填：
   ```
   https://qnlciqqimwzlsvmplskq.supabase.co/auth/v1/callback
   ```
6. 保存后，会看到 **Client ID** 和 **Client Secret**，复制下来。

### 4.2 在 Supabase 里

1. **Authentication** → **Providers** → 找到 **Google**。
2. 把 Google 开关打开（Enable）。
3. 把 **Client ID**、**Client Secret** 粘贴进去，保存。

### 4.3 自测

打开你的登录页，点「Google」，能跳到 Google 选账号并成功回到你站内首页，即通过。

---

## 第五步：配置 Apple 登录（要用 Apple 才做）

### 5.1 在 Apple 后台

1. 打开 **https://developer.apple.com/**，登录。
2. **Certificates, Identifiers & Profiles** → **Identifiers**。
3. 新建或选用一个 **App ID**，并确保勾选了 **Sign in with Apple**。
4. 在 **Services IDs** 里新建一个 Services ID（用于网页 Sign in with Apple），在它的 **Sign in with Apple** 配置里：
   - **Return URL** 填：
     ```
     https://qnlciqqimwzlsvmplskq.supabase.co/auth/v1/callback
     ```
5. 在 **Keys** 里新建一个 Key，勾选 **Sign in with Apple**，下载 `.p8` 文件，记下 **Key ID**、**Team ID**、**Services ID**（和 Bundle ID 等）。用 Apple 提供的工具或脚本用这个 Key 生成一个 **Secret（JWT）**，Supabase 里要填。

### 5.2 在 Supabase 里

1. **Authentication** → **Providers** → **Apple**。
2. 打开 Apple 开关，按 Supabase 表单要求填写：
   - **Services ID**、**Secret（JWT）**、**Team ID**、**Key ID** 等（和 Apple 后台一致）。
3. 保存。

### 5.3 自测

在登录页点「Apple」，能跳转 Apple 并成功回到站内首页即通过。

---

## 第六步：配置 Facebook 登录（要用 Facebook 才做）

### 6.1 在 Facebook 后台

1. 打开 **https://developers.facebook.com/** → 我的应用 → 选应用（或新建）。
2. 产品里添加 **Facebook 登录** → 选 **网页**。
3. 在 **Facebook 登录 → 设置** 里，**有效 OAuth 重新導向 URI** 添加：
   ```
   https://qnlciqqimwzlsvmplskq.supabase.co/auth/v1/callback
   ```
4. **设置 → 基本** 里抄下 **应用程式編號（App ID）**、**應用程式密鑰（App Secret）**。

### 6.2 在 Supabase 里

1. **Authentication** → **Providers** → **Facebook**。
2. 打开开关，把 **App ID**、**App Secret** 贴进去，保存。

### 6.3 自测

点「Facebook」能跳转并成功回到站内即通过。

---

## 第七步：配置 X（Twitter）登录（要用 X 才做）

### 7.1 在 X 后台

1. 打开 **https://developer.twitter.com/** → 项目与应用。
2. 在 **User authentication settings** 里配置 OAuth 2.0，**Callback URI / Redirect URL** 填：
   ```
   https://qnlciqqimwzlsvmplskq.supabase.co/auth/v1/callback
   ```
3. 抄下 **Client ID**、**Client Secret**（OAuth 2.0）。

### 7.2 在 Supabase 里

1. **Authentication** → **Providers** → **Twitter**。
2. 打开开关，把 **Client ID**、**Client Secret** 贴进去（若 Supabase 写的是 API Key / API Secret，就对应填 OAuth 2.0 的 Client ID / Client Secret），保存。

### 7.3 自测

点「X」能跳转并成功回到站内即通过。

---

## 第八步：邮箱密码登录（一般不用改）

1. 在 Supabase：**Authentication** → **Providers** → **Email**。
2. 确认是 **开启** 状态即可（默认就是开的）。
3. 自测：用邮箱注册/登录一次，能收到邮件并完成登录即通过。

---

## 最终核对（确保百分百能登录）

在正式对外前，按顺序确认：

| 序号 | 检查项 | 你做了吗 |
|------|--------|----------|
| 1 | Supabase **URL Configuration** 里：Site URL = `https://mycelestial.app`，Redirect URLs 里有 `https://mycelestial.app/**` 和两个 localhost | ☐ |
| 2 | Google/Apple/Facebook/X 后台填的「回調 / Redirect URI」都是 `https://qnlciqqimwzlsvmplskq.supabase.co/auth/v1/callback`（不是 mycelestial.app） | ☐ |
| 3 | 要用的登录方式在 Supabase **Providers** 里都是「开启」，且 ID/Secret 都填了 | ☐ |
| 4 | 本地和部署环境都有 `VITE_SUPABASE_URL`、`VITE_SUPABASE_PUBLISHABLE_KEY` | ☐ |
| 5 | 用无痕窗口打开登录页，Google、Apple（或 Facebook、X）各测一次，都能跳转并回到站内首页 | ☐ |

全部打勾后，这些入口就是「一定能登录」的状态。

---

## 常见问题

- **点了 Google/Apple 没反应或报错**  
  先检查：Supabase 的 **Redirect URLs** 是否已加 `https://mycelestial.app/**` 和本地用的 `http://localhost:8080/**`、`http://localhost:5173/**`。

- **跳转到 Google/Apple 后回不来或报错**  
  检查：该平台后台的「回調 / Redirect URI」是否**一字不差**为 `https://qnlciqqimwzlsvmplskq.supabase.co/auth/v1/callback`。

- **本地能登、线上不能登（或反过来）**  
  检查：对应环境的 **环境变量** 是否填了 `VITE_SUPABASE_URL` 和 `VITE_SUPABASE_PUBLISHABLE_KEY`，以及 Supabase 的 Redirect URLs 是否包含该环境的域名（线上用 `https://mycelestial.app/**`，本地用 localhost）。
