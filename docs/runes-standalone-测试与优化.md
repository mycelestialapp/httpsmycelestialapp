## 二、测试说明

### 2.1 如何运行

1. **本地运行**  
   - 用浏览器直接打开 `docs/runes-standalone.html`（双击或拖入浏览器）。  
   - 或在本项目根目录执行：`npx serve .` 或 `python -m http.server 8000`，然后访问 `http://localhost:8000/docs/runes-standalone.html`。

2. **依赖**  
   - 无任何外部依赖（无 CDN、无字体/图片外链）。  
   - 仅依赖现代浏览器的 ES5 与 Canvas API。

### 2.2 预期效果

- **单符指引**：点击后出现 1 张符文石，约 0.4s 后翻转显示中文名与正/逆位释义；若有粒子，会播放极光粒子。
- **三符命运**：3 张石头依次翻转（间隔约 0.3s）；解读区仍为空，需再点「唤醒解读」。
- **五符十字**：5 张石头同理。
- **快速三符（短路径）**：直接抽 3 符并约 1.5s 后自动填充「唤醒解读」四层内容（全景镜像 / 阴影网络 / 动力链条 / 转化锚点）。
- **唤醒解读**：若已抽符，则静态生成上述四层解读；若未抽符，提示「请先投掷符文」。
- **无障碍**：系统开启「减少动态效果」时，翻转动画缩短、粒子不启动；`prefers-reduced-motion` 生效。
- **性能**：切到其他标签页时，粒子动画通过 `visibilitychange` 与 `cancelAnimationFrame` 停止；返回页面后若已抽符会重新轻量播放粒子。

### 2.3 潜在问题及修复

| 现象 | 可能原因 | 修复 |
|------|----------|------|
| 符文符号显示为方框 | 系统/浏览器缺少北欧符文字体 | 符文已用 Unicode 码位（如 U+16A0），多数现代系统可显示；若仍异常可考虑用 SVG/图片替代（需自行扩展）。 |
| 点击「唤醒解读」无反应 | 未先抽符 | 先点「单符指引」「三符命运」或「五符十字」再点「唤醒解读」。 |
| 粒子不出现 | 系统开启「减少动态效果」 | 属预期行为；或页面在后台（`document.hidden`）时已暂停。 |
| 移动端石头过小/过大 | 视口单位差异 | 已用 `min(140px, 28vw)` 等做响应式，可据实机再微调 `#rune-area` 的 gap 与 .rune-stone 宽高。 |

## 三、优化建议（不自动实现，仅供产品迭代）

1. **音效**  
   在翻转或「唤醒解读」时增加可选的环境音（如火焰噼啪、风铃），并随系统「静音」或用户设置关闭，以增强仪式感。

2. **Vue/React 重构**  
   将数据层（`elderFuthark`、`getReadingPayload`）、核心逻辑（`castRunes`、`generateMasterReading`）与视图分离，便于接入现有 App 路由（如 `/oracle/reading?tool=runes`）与状态管理，并复用设计令牌（如 `--rune-gold`）。

3. **后端 AI 解读**  
   用 `getReadingPayload()` 的 JSON 调用后端接口，返回四层解读文案并渲染；前端保留静态解读作为降级或加载占位，以提升「大师级」体验。

4. **多语言**  
   将 `elderFuthark` 的 `cn`、`upright`、`rev` 及四层解读模板抽成 i18n 键，便于后续中/英/繁等多语言与无障碍朗读。

5. **历史与分享**  
   将每次 `getReadingPayload()` 与生成结果存到 `localStorage` 或上传到账户，并提供「保存为图片」或分享链接，以增强留存与传播。
