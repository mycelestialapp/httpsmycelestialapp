# 占卜/占星产品策划 · 主流参考方案

> 参考：爱占星、星座大师、Co–Star、The Pattern、Sanctuary、CHANI、Stardust、今日星座、测测等主流产品，结合本产品现有能力（星图、灵魂原型、八字、塔罗、关系页、节奏/运势提示）整理。

---

## 策划一：日/周/月运势体系（高优先级）

### 参考
- 爱占星、每日星座运势、Co–Star：日运/周运/月运/年运是标配。
- Co–Star：每日推送一句「今日星象」+ 毒舌/娱乐化语气，打开率极高。

### 目标
把「看运势」做成每日习惯，而不是偶尔打开。

### 功能要点
| 模块 | 内容 | 实现思路 |
|------|------|----------|
| **日运** | 每日一段话（爱情/事业/健康/财运任选或轮换） | 复用现有 `cosmicTip*` / 季节提示，按「日期 + 太阳星座」生成或选取模板；可加「今日宜/忌」短句。 |
| **周运** | 一周主题 + 3～5 条要点 | 按周数 + 星座生成，存 i18n 或后端；每周一更新。 |
| **月运** | 本月重点领域 + 建议 | 按月 + 星座，可与「季节提示」合并或替换。 |
| **年运** | 年度主题（可选） | 年初上线，按生肖/星座生成一条年度关键词。 |

### 呈现位置
- 星图首页顶部：今日一句话 + 「查看完整日运」入口。
- 节奏/运势页（Rhythm）：日/周/月 Tab，与现有节奏内容整合。
- 推送（若后续做）：每日 8:00 左右推送「今日星象一句话」。

### 差异化
- 语气可偏「清醒、自省」（类似 The Pattern）或「毒舌、好玩」（类似 Co–Star），与灵魂原型文案统一。
- 若保留五行：日运可带一句「今日五行小提示」（如：水日，宜静）。

## 策划二：星座/关系配对与「合盘」轻量化

### 参考
- 今日星座、测测、Co–Star：星座配对、缘分匹配、好友/伴侣合盘是刚需。
- Co–Star：添加好友后对比两人星盘，给出兼容度与一句话解读。

### 目标
把「我和 ta 合不合」做成可分享、可传播的入口，并自然引导到深度内容（灵魂原型、星盘）。

### 功能要点
| 模块 | 内容 | 实现思路 |
|
|------|------|----------|
| **星座配对** | 选两个太阳星座 → 匹配度 % + 一段话（爱情/友情/合作） | 12×12 矩阵预置文案，i18n；可加「最佳组合」「需磨合」标签。 |
| **双人合盘（轻量）** | 两人出生日期 → 太阳/月亮星座对比 + 一句话 | 用现有 `getSunSignFromDate`，月亮可用简化规则或暂用「太阳合盘」；输出 1 屏内结论。 |
| **分享与拉新** | 「我 vs 某某」结果页可生成图或链接 | 结果页带「分享给 ta」链接（含 ref/soul 参数），对方打开后可填自己的生日看「对方视角」，形成双端使用。 |

### 呈现位置
- 关系页（Relationships）：现有关系列表上增加「星座配对」「合盘」入口。
- 星图/灵魂原型页：增加「看看和 ta 的契合度」入口，跳转配对或合盘。

### 与现有能力结合
- 已有 `RelationshipsPage`、`matchingEngine`、ref/soul 参数，可在此基础上加「按星座/生日的配对结果」与分享链接。

## 策划三：「每日一签」与仪式感（日活/留存）

### 参考
- 星空塔罗、今日星座：每日一牌、每日一句。
- CHANI：占星 + 冥想/肯定语，把「预测」和「应对」绑在一起，提升订阅价值。

### 目标
每天有一个固定理由打开 App（抽牌/抽签/看一句话），并可选延伸为「应对今日」的轻量内容。

### 功能要点
| 模块 | 内容 | 实现思路 |
|------|------|----------|
| **每日一签** | 打开 App 或进入某页时展示一张牌/一句签文 | 用「日期种子」保证同一天同一用户结果一致；塔罗 78 张或 22 张大牌 + 简短解读；或复用现有每日智慧/金句。 |
| **每日一牌** | 塔罗单张 + 正/逆位 + 2～3 句解读 | 同上，按日期种子选牌；可带「今日启示」「宜/忌」短句。 |
| **仪式感动效** | 抽签/翻牌动画、音效（可选） | 提升分享欲与记忆点；可做「今日签文」分享图。 |

### 呈现位置
- 首页（星图）顶部或弹层：首次进入当日显示「今日一签」；或独立 Tab「每日」。
- 与 DailyCheckin / DailyWisdom 整合：签到后展示今日签/今日牌。

### 商业化
- 免费：每日一签/一牌 + 简短解读。
- 会员/付费：解锁「完整牌义」「本周牌阵」「历史签文」等。

## 策划四：能量地圖/五行重構（解决「感觉不好」）

### 参考
- 主流 App 较少单独做「五行平衡分析」；多数以星座/星盘为主，东方术数作为补充（如八字、紫微）。
- CHANI / The Pattern：强调「了解自己」与「心理模式」，而非「你缺什么要补什么」。

### 目标
保留「基于生日的五行」能力，但从「体检式平衡」改为「气质/倾向描述」，减少被评判感。

### 方案 A：气质化 + 与星座联动
| 改动 | 说明 |
|------|------|
| 命名 | 「能量地圖」→「五行氣質」或「本命五行」；「Balance Analysis」→「五行分布」或收起。 |
| 文案 | 主导元素：「你的主調是金/土…」；最弱元素：「水較少顯現，可多留意流動與休息」；避免「弱、不足、需補」。 |
| 与灵魂原型联动 | 在灵魂原型卡旁或下方加一句：「你的太陽星座屬火象，五行偏金土，形成結構感與行動力。」 |
| 天機解語 | 改为「對你這一型的建議」：一句主調 + 一句可選的滋養方向，不列「強/弱/補」清單。 |

### 方案 B：精简呈现
| 改动 | 说明 |
|------|------|
| 只保留雷達圖 | 去掉下方整块条形「Balance Analysis」，或收成一行「金 100% · 土 94% · 水 35%…」。 |
| 天機解語缩短 | 一句话：主调 + 一个可选方向；或与「今日五行小提示」合并。 |

### 方案 C：可选隐藏
- 设置中增加「不显示五行/能量地图」，仅保留星图与灵魂原型，满足「只信星座」的用户。

## 策划五：推送与订阅（日活 + 变现）

### 参考
- Co–Star：每日推送一句今日星象，语气毒舌，打开率高。
- CHANI / Stardust：订阅制（约 $10–12/月），解锁深度内容、冥想、周期追踪等。

### 目标
用「每日一条推送」拉回用户；用「会员/订阅」承载深度内容与增值服务。

### 功能要点
| 模块 | 内容 | 实现思路 |
|------|------|----------|
| **推送** | 每日 1 条：今日星象/今日签/今日宜忌 | 按用户太阳星座（需存生日）+ 日期生成；可 A/B 语气（毒舌 vs 治愈）。 |
| **订阅权益** | 日运/周运/月运完整版、合盘详细报告、历史签文、去广告等 | 与现有 Subscription 页打通；可设「免费日运一句 + 付费完整段」。 |
| **节奏** | 推送时间可配置（如 8:00）；新用户引导时请求通知权限。 | 需后端/推送服务（如 Firebase、OneSignal）及订阅状态。 |

### 注意
- 推送需用户授权，文案需避免「迷信/医疗/投资建议」等合规风险，以娱乐、自我觉察为主。

## 策划六：内容矩阵与「主题日」（运营向）

### 参考
- The Pattern：Cosmic Climate（集体周期）、音频合集、主题月。
- 爱占星：多体系（现代/古典/印度/八字/紫微）满足不同偏好。

### 目标
让「每天/每周都有新东西可看」，而不只有静态星盘与灵魂原型。

### 功能要点
| 模块 | 内容 | 实现思路 |
|------|------|----------|
| **主题日** | 周一爱情、周二事业、周三健康… 或 月相日、节气日 | 日运/推荐内容按星期或节气切换主题；i18n 预置或配置。 |
| **月相/节气** | 新月、满月、节气当日特殊提示或短文案 | 按公历计算月相与节气，当日展示一句「新月宜许愿」类文案。 |
| **专题** | 如「水逆指南」「星座年度回顾」 | 运营位入口，链接到长文或合集；可会员专享。 |

### 呈现位置
- 星图首页或 Rhythm 页：今日主题标签 + 对应短内容；节气/月相在日历或 Banner 提示。

## 策划七：社交与传播（拉新/裂变）

### 参考
- Co–Star：加好友、对比星盘、分享到社交平台。
- 测测/今日星座：配对结果、测试结果分享图。

### 目标
让「算完/看完」能变成「发给别人」的素材，带 ref 或邀请码。

### 功能要点
| 模块 | 内容 | 实现思路 |
|------|------|----------|
| **分享图** | 灵魂原型金句、今日签、配对结果生成图 | 前端 Canvas 或服务端生成图，带 Logo/二维码；保存或分享到微信/微博。 |
| **邀请/链接** | 「看看我们合不合」链接带 soul= 或 ref= | 已有 ref/soul；在配对/合盘结果页强化「分享给 ta」按钮。 |
| **轻量社交** | 匿名「今日同星座的人都在看什么」或「同星座投票」 | 需后端统计与展示，可选做。 |

## 优先级与落地建议

| 优先级 | 策划 | 理由 | 建议落地顺序 |
|--------|------|------|----------------|
| P0 | 策划四（能量地圖重构） | 你已明确「感觉不好」，先解决体验与语义。 | 先做方案 A 或 B，再考虑 C。 |
| P1 | 策划一（日/周/月运势） | 主流标配，且你已有 cosmicTip/季节提示可复用。 | 先做「今日一句话」+ Rhythm 页日/周/月结构。 |
| P1 | 策划二（配对/合盘） | 关系页与 matching 已有基础，配对易出传播。 | 先做星座配对 12×12，再做轻量合盘。 |
| P2 | 策划三（每日一签） | 提升日活与打开习惯，可与日运合并。 | 与策划一一起做「今日内容」区块。 |
| P2 | 策划五（推送与订阅） | 依赖推送与付费基建，中长期变现。 | 在日运稳定后加推送；订阅与现有订阅页打通。 |
| P3 | 策划六、七 | 内容运营与社交，可持续迭代。 | 按运营节奏加主题日、分享图、邀请链接。 |

## 与现有代码的对应关系（便于开发）

- **日/周/月运、今日一签**：可复用 `cosmicTip*`、`seasonThemeHintSet*`、`DailyWisdom`、`DailyCheckin`；新键可放在 `oracle` 或新命名空间下。
- **星座配对**：可新增 i18n 键如 `oracle.pairing.aries.taurus` 等；Relationships 页或星图页入口。
- **合盘**：`getSunSignFromDate` 已有；需月亮星座时可扩展或先用太阳。
- **能量地圖**：改 `EnergyRadar` 文案与展示（方案 A/B/C），必要时在设置中加「隐藏五行」。
- **推送**：需接入推送 SDK 与后端存储生日/星座；订阅状态与现有订阅逻辑一致即可。

如需把某一则策划拆成「需求列表 + 接口/文案表」，可以指定策划编号，我按该则细化到可直接开发的级别。
