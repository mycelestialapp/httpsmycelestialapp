# 塔羅板塊 · 按「沉浸療癒 + 行動導向」方案整改清單

> 對照《塔羅 App 界面設計方案》，列出需改動的**具體位置、檔案與優先級**。現有架構：首頁 OraclePage → 點塔羅 → OracleReadingPage?tool=tarot → OracleSanctumEntrance + TarotPanel。

---

## 一、整改總覽（按模塊）

| 模塊 | 對應方案步驟 | 主要檔案 | 優先級 | 狀態 |
|------|--------------|----------|--------|------|
| 全局導航與主題 | 步驟 1（導航架構 + 顏色） | BottomNav、index.css、tailwind、AppLayout | P2 | 可選 |

| 抽牌過程頁 | 步驟 3（沉浸全屏、牌堆、翻牌） | TarotPanel.tsx、OracleReadingPage.tsx | P1 | 待加強 |
| 解牌結果頁 | 步驟 4（問題重述 + 牌 + 總結 + 按鈕） | TarotPanel.tsx、MasterOracleReadingView.tsx | P1 | 已有，可微調 |

## 二、分項整改清單

### 1. 塔羅入口（OracleSanctumEntrance.tsx）— 步驟 2

| 序號 | 整改項 | 方案要求 | 現狀 | 具體改動 |
|------|--------|----------|------|----------|
| 2.1 | **免責 Banner** | 頂部固定、可關閉（localStorage 不再顯示）、中英雙語 | 已有固定免責區塊，無關閉鈕、無英文 | ① 改為可關閉 Banner（小 X），點擊後 `localStorage.setItem('tarot_disclaimer_seen', '1')`，下次不顯示；② 文案加英文句（或 i18n 鍵） |
| 2.2 | **每日洞察輪播** | 今日一張牌 + 一句行動建議，點擊進完整閱讀 | 無 | 在「免責」與「Step 1」之間加一區：今日一牌預覽（可從 DailyOneCard 或新接口取），點擊跳轉今日一牌完整頁或彈窗 |
| 2.3 | **問題輸入框** | 大輸入框、placeholder 即時引導 | 已有 placeholder「例如：我該如何改善目前的感情溝通？」 | 可選：placeholder 依當前選中分類切換（感情/事業各一句）；或維持一句即可 |
| 2.4 | **「優化問題」按鈕** | 輸入框旁，點擊喚起改寫助手 | 無 | 在「或自己輸入問題」展開區內，輸入框右側或下方加按鈕「優化問題」；點擊後可先 toast「即將上線」或接 API 彈出改寫建議 + 一鍵替換 |
| 2.5 | **分類 Chips** | 感情／事業／學業／健康／靈性／其他，橫向滾動 | 已是橫向按鈕（flex-wrap），分類齊全 | 可選：改為橫向滾動條（overflow-x-auto），或加圖標（💕💼📚🌿✨） |
| 2.6 | **開始抽牌按鈕** | 大圓形浮動、紫金漸層、「開始閱讀」 | 目前為「選問題 → 選牌陣 → 直接進入抽牌」，無獨立「開始閱讀」大按鈕 | 可選：在「當前問題」區塊下方加大按鈕「開始閱讀」/「Draw Cards」，點擊後捲動到 TarotPanel 或聚焦選牌陣區 |

### 2. 抽牌過程（TarotPanel.tsx + OracleReadingPage）— 步驟 3

| 序號 | 整改項 | 方案要求 | 現狀 | 具體改動 |
|------|--------|----------|------|----------|
| 3.1 | **沉浸全屏** | 抽牌時隱藏導航條 | 整頁為 OracleReadingPage，底部導航仍在 | 抽牌中（step 為 shuffling/draw）時，OracleReadingPage 內用 class 或 state 隱藏 BottomNav（或全屏遮罩），結束後恢復 |
| 3.2 | **牌堆視覺** | 78 張牌背堆疊、輕微晃動/發光 | 現有牌背樣式，動畫較少 | 洗牌/抽牌前增加牌堆容器（多張牌背疊放）+ CSS/framer-motion 輕微晃動或光暈 |
| 3.3 | **抽牌手勢** | 支援 Swipe up / Tap / Shake | 目前為點擊抽牌 | 可選：加「滑動抽牌」或保持點擊，Shake 可列路線圖 |
| 3.4 | **翻牌動畫** | 3D 翻轉（背面→正面） | 有翻牌邏輯，是否 3D 視實現而定 | 用 framer-motion 的 rotateY 做翻轉，或維持現有動畫 |
| 3.5 | **多張牌陣佈局** | 過去-現在-未來橫排等 | 已有三張/五張等牌陣，橫向排列 | 確認響應式與間距，可加輕微浮動陰影 |

### 3. 解牌結果（TarotPanel + MasterOracleReadingView）— 步驟 4

| 序號 | 整改項 | 方案要求 | 現狀 | 具體改動 |
|------|--------|----------|------|----------|
| 4.1 | **頂部問題重述** | 牌陣名稱 + 問題 | 鏡像復述「你正在問的是：…」、靈魂核心、牌面解讀、牌陣總結 | 保持即可，必要時把「問題」更突出（字體或區塊標題） |
| 4.2 | **牌面大圖** | 可左右滑動、每張下牌名+正逆+領域解讀 | 已有牌面展示、ReadingBlock 展開有解讀 | 可選：結果區頂部加牌面橫向滑動（swiper）；單張下方已有牌名+正逆+愛情/事業等 |
| 4.3 | **總結與行動指引** | 底部粗體重點、「你可以試著…」 | 牌陣總結已有收束句、靈魂核心有指引 | 可選：在牌陣總結下再加一句「本週可試著：…」（從 microAction 或建議句來） |
| 4.4 | **按鈕列** | 保存、重新抽牌、分享、相關學習 | 有存解讀、有分享相關；重新抽牌需返回或重置 | ① 結果區底部固定「保存到日誌」「重新抽牌」「分享」；② 「相關學習」可連到卡義瀏覽（若無則預留） |
| 4.5 | **視覺** | 牌卡柔光邊框、輕浮動陰影 | 已有圓角邊框與金色系 | 可加 `shadow-lg`、`ring-1 ring-gold-strong/20` 等 Tailwind 類 |

### 6. 日誌與設定（步驟 5）

| 序號 | 整改項 | 方案要求 | 現狀 | 具體改動 |
|------|--------|----------|------|----------|
| 6.1 | **日誌列表** | 時間線、標籤（感情/事業） | ReadingHistoryPage 已有解讀列表 | 可選：篩選「塔羅」、標籤顯示分類；時間線樣式微調 |
| 6.2 | **設定：語言** | 繁/簡/英 | 已有 LanguageSwitcher | 確認繁簡英都有 |
| 6.3 | **設定：暗/亮模式** | 暗模式默認，可切換 | 已有主題切換（多款主題） | 可選：強調「深色」為推薦，或維持現有 |
| 6.4 | **設定：逆位開關** | 抽牌是否出現逆位 | 無 | 在 SettingsPage 加「塔羅逆位」開關，存 localStorage；TarotPanel 抽牌時讀取該設定 |
| 6.5 | **設定：免責重讀** | 可再次查看免責文案 | 無 | 設定頁加「塔羅免責聲明」條目，點擊彈出與入口相同的免責文 |
| 6.6 | **設定：數據刪除** | 合規用 | 若有帳號/本地存儲 | 隱私頁或設定頁提供「刪除我的解讀記錄」或「清除本地數據」，並說明範圍 |

## 三、建議實施順序（不影響現有功能）

1. **P0（先做）**  
   - 2.1 免責可關閉 + 可選英文  
   - 2.4 「優化問題」按鈕（可先接 toast 或靜態建議）  
   - 2.6 可選：大按鈕「開始閱讀」  

2. **P1（短期）**  
   - 3.1 抽牌時隱藏導航 / 全屏感  
   - 3.2 牌堆視覺、3.4 翻牌動畫  
   - 4.2 解牌結果區牌面橫滑（可選）  
   - 4.4 結果頁按鈕列補齊  
   - 6.4 逆位開關、6.5 免責重讀、6.6 數據刪除  

3. **P2（中後期）**  
   - 2.2 每日洞察輪播  
   - 5.1 首頁今日塔羅一牌、5.2 今日推薦問題  
   - 1.2 塔羅/全局主色微調  
   - 6.1 日誌時間線與標籤優化
