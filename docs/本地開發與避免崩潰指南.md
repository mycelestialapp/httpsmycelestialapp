# 本地開發與避免崩潰指南

本指南教你：如何啟動專案、在瀏覽器裡打開網頁，以及如何減少 Cursor 記憶體不足（OOM）和網頁連不上的情況。

---

## 一、用「一鍵腳本」啟動開發伺服器（推薦）

### 第 1 步：找到腳本

1. 打開 **檔案總管**（Windows 鍵 + E）。
2. 在左側或地址欄輸入：`d:\Dmy-projects`，按 Enter。
3. 在資料夾裡找到 **`start-dev.bat`**（圖示像一個齒輪或視窗）。

### 第 2 步：雙擊執行

1. **雙擊** `start-dev.bat`。
2. 會彈出一個**黑色視窗**（命令提示字元），裡面會跑很多字。
3. **不要關閉這個黑視窗**，關掉就會停止伺服器，網頁就打不開了。

### 第 3 步：看終端顯示的網址

等幾秒，黑視窗裡會出現類似：

```
  VITE v5.x.x  ready in xxx ms
  ➜  Local:   http://localhost:8080/
  或
  ➜  Local:   http://localhost:8081/
```

- 如果 8080 被佔用，會自動改成 **8081**。
- 記住你看到的網址（**8080** 或 **8081**）。

### 第 4 步：用瀏覽器打開

1. 打開 **Chrome** 或 **Edge**（不要用 Cursor 內建的「Browser Tab」）。
2. 在網址列輸入：`http://localhost:8080` 或 `http://localhost:8081`（和終端裡一致）。
3. 按 Enter，就能看到你的專案頁面。

### 第 5 步：之後要關機或不用時

- 回到那個**黑色視窗**，點一下視窗，按鍵盤 **Ctrl + C**，或直接關閉視窗，開發伺服器就會停止。
- 下次要用時，再雙擊 `start-dev.bat` 即可。

---

## 二、不用腳本、手動用終端啟動（進階）

如果你想自己打指令：

1. 按 **Windows 鍵**，輸入 **PowerShell** 或 **cmd**，按 Enter 打開終端。
2. 輸入下面兩行（可複製貼上），每行結束按 Enter：

   ```powershell
   cd d:\Dmy-projects
   npm run dev
   ```

3. 出現 `Local: http://localhost:8080`（或 8081）後，用瀏覽器打開該網址。
4. 要停止時，在終端按 **Ctrl + C**。

---

## 三、為什麼網頁會「連不上」或「刷新不出來」？

常見原因：

| 情況 | 原因 | 對應做法 |
|------|------|----------|
| 顯示 **Connection Failed / ERR_CONNECTION_REFUSED** | 開發伺服器沒在跑 | 雙擊 `start-dev.bat` 或手動執行 `npm run dev`，再重新整理瀏覽器。 |
| 用 Cursor 內建瀏覽器開 localhost | 內建瀏覽器容易和 Cursor 一起崩潰或斷線 | 改用 **Chrome / Edge** 開 `http://localhost:8080`（或 8081）。 |
| Cursor 視窗崩潰（OOM 記憶體不足） | 專案大、Agent、內建瀏覽器同時吃很多記憶體 | 照下面「四」的做法減輕負擔。 |

---

## 四、如何減少 Cursor「記憶體不足」崩潰？

1. **少用 Cursor 內建瀏覽器**  
   用本機 Chrome/Edge 開 localhost，關掉 Cursor 裡不需要的 Browser Tab。

2. **一次只開一個 Agent**  
   不要同時開很多個 Agent 或一次搜尋/改動整個專案。

3. **關掉不用的分頁與專案**  
   只保留當前在編輯的檔案，其它專案或大檔案先關掉。

4. **開發伺服器在「外面」跑**  
   用雙擊 `start-dev.bat` 或在外面的 PowerShell 跑 `npm run dev`，不要只依賴 Cursor 內建終端。這樣 Cursor 關掉或崩潰時，伺服器有機會還在跑，你還能用瀏覽器開網頁。

5. **重開 Cursor 後**  
   若網頁又連不上，先看黑視窗（或 PowerShell）裡的 dev 是否還在跑；若已關閉，再雙擊一次 `start-dev.bat` 即可。

## 六、出錯時可以這樣檢查

1. **網頁顯示連線失敗**  
   → 看 `start-dev.bat` 的黑視窗是否還在、有沒有報錯。若已關閉，重新雙擊 `start-dev.bat`。

2. **改了程式但網頁沒變**  
   → 在瀏覽器按 **Ctrl + F5** 強制重新整理；或關掉該分頁再重新打開 localhost。

3. **埠號不是 8080**  
   → 看終端寫的是 8080 還是 8081，瀏覽器網址要一致。

4. **Cursor 又 OOM 崩潰**  
   → 先不要用內建瀏覽器，用本機瀏覽器 + 外面跑的 dev；必要時重開 Cursor，再雙擊 `start-dev.bat` 確認伺服器有起來。

按上面步驟做，就能穩定在本地打開和刷新專案網頁，並減少 Cursor 崩潰。若某一步卡住，可以把畫面或錯誤訊息截圖，再問具體哪一步不會操作。
