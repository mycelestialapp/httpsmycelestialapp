# 西方占卜 · 星圖板塊內容生成方案

依據**西方占卜邏輯**與**西方主流 App（Co-Star、The Pattern、Sanctuary 等）**的標準，說明我們應如何生成星圖板塊的內容、依賴什麼數據與邏輯。

---

## 一、西方主流 App 的通用標準

| 維度 | 主流做法 | 說明 |
|------|----------|------|
| **輸入數據** | 出生日期 + 出生時間 + 出生地 | 時間與地點用於上升星座、宮位、精確月亮與行星位置；僅有日期時可先做太陽星座。 |
| **本命解讀** | 太陽 + 月亮 + 上升（及行星、宮位） | 性格／靈魂原型來自「太陽星座」為核心，再疊加月亮（情緒）、上升（外在表現）。 |
| **每日內容** | 當下星象（過境、月相）與本命盤的關係 | 每日一句／短建議多與「今日月亮星座、月相、或重要過境」有關。 |
| **年度／階段** | 太陽回歸、年運、或重要過境週期 | 「這一程」類的主題常來自年度星象或個人年運，而非固定一句話。 |
| **文案來源** | 預設模板 × 星座／星象標籤，或 NLG／AI 輔助 | 先做「星座 + 情境 → 固定文案」即可；進階再考慮算法生成。 |

結論：**內容應由「本命盤（至少太陽，進階太陽+月亮+上升）」與「當下星象（日/月/過境）」共同驅動，而不是由八字五行驅動。**

---

## 二、我們應採用的數據分級

根據用戶能提供的數據，分三檔，每檔能算出的內容不同：

| 檔位 | 用戶提供的數據 | 可計算的西方占星內容 | 可生成的內容 |
|------|----------------|----------------------|--------------|
| **L1 僅日期** | 年、月、日 | **太陽星座**（Sun sign） | 靈魂原型／性格描述、金句、能量關鍵詞 → 全部改為「依太陽星座」 |
| **L2 日期+時間** | 年、月、日 + 出生時間 | 太陽星座 + **月亮星座**（Moon sign，需星曆表） | 在 L1 基礎上增加「情緒／內在」維度，可做更細的標籤或副標題 |
| **L3 日期+時間+地點** | 年、月、日 + 時間 + 出生地（經緯度） | 太陽 + 月亮 + **上升星座**（Rising）+ 宮位、行星落宮 | 與 Co-Star／The Pattern 同級：完整本命盤、過境、合盤等 |

**建議**：  
- 第一階段先做 **L1（僅日期）**：用出生日期算出**太陽星座**，所有「靈魂原型、金句、能量關鍵詞」改為依太陽星座生成。  
- 第二階段再接入**出生時間**（及可選出生地），引入月亮、上升與更多內容。

## 三、內容類型與生成邏輯對照表

| 內容類型 | 當前邏輯（東方） | 應改為（西方） | 依據數據 |
|----------|------------------|----------------|----------|
| **靈魂原型名稱** | 五行主元素 → 火/水/木/金/土 | **太陽星座** → 12 星座標籤（如「獅子座的靈魂」「雙魚的夢境」） | 出生月、日 |
| **能量關鍵詞** | 五行對應關鍵詞 | **太陽星座**對應的關鍵詞（每星座一組） | 出生月、日 |
| **核心理念 / Tagline** | 五行主元素對應一句話 | **太陽星座**對應一句「被看見」的標語 | 出生月、日 |
| **金句** | 五行主元素對應金句 | **太陽星座**對應一句可分享金句 | 出生月、日 |
| **今日宇宙提示** | 星期 1～7 輪播固定句 | **月相**或**當日月亮星座**或「太陽星座 × 今日月亮」組合 → 選一句短提示 | 當前日期（+ 可選出生月日做個人化） |
| **這一程 · 年度主題** | 靜態一句話 | **當年星象**（如木土星象、年度日月食）或「太陽星座 × 年」的簡化年運主題 | 當前年（+ 出生月日） |
| **工作 / 感情 / 自己** | 靜態三句 | 由「年度主題」衍生三句，或「太陽星座 × 年」各一條建議 | 當前年 + 出生月日 |

靈魂原型整塊：**全部改為依太陽星座**；今日提示與「這一程」：**依當下星象（或月相/月亮星座）+ 可選太陽星座**。

## 四、太陽星座的計算邏輯（L1 僅日期）

採用**熱帶黃道 12 星座**，僅用**月、日**即可得到太陽星座（年用於處理閏年邊界，常規實現可忽略）：

| 星座（英文） | 常見日期區間（月.日） |
|--------------|------------------------|
| Capricorn 摩羯 | 12.22 － 1.19 |
| Aquarius 水瓶 | 1.20 － 2.18 |
| Pisces 雙魚 | 2.19 － 3.20 |
| Aries 牡羊 | 3.21 － 4.19 |
| Taurus 金牛 | 4.20 － 5.20 |
| Gemini 雙子 | 5.21 － 6.20 |
| Cancer 巨蟹 | 6.21 － 7.22 |
| Leo 獅子 | 7.23 － 8.22 |
| Virgo 處女 | 8.23 － 9.22 |
| Libra 天秤 | 9.23 － 10.22 |
| Scorpio 天蠍 | 10.23 － 11.21 |
| Sagittarius 射手 | 11.22 － 12.21 |

邊界日（如 3.20 / 3.21）可能因年份而落在不同星座，進階可接星曆表；第一版可用固定邊界。

**產出**：  
- 輸入：`year, month, day`  
- 輸出：`sunSign: 'leo' | 'virgo' | ...`  
- 所有「靈魂原型、關鍵詞、Tagline、金句」改為按 `sunSign` 從預設文案表讀取。

## 五、今日宇宙提示的西方邏輯（可選進階）

- **方案 A（最簡）**：依**月相**（新月、上弦、滿月、下弦等）或**當日月亮星座**輪播 7～30 條固定句，不區分用戶。  
- **方案 B（個人化）**：依「**用戶太陽星座 + 當日月亮星座**」或「太陽星座 + 月相」組合，每組合對應一句短提示（預設表），更貼近主流 App 的「每日個人化」。

第一階段可先做方案 A（月相或星期輪播），第二階段再改為方案 B。

---

## 六、這一程 · 年度主題的西方邏輯（可選進階）

- **方案 A（最簡）**：依**當前年份**固定幾條年度主題（如「木星在 XX 星座的年份」），所有人同年同一句，或再按**太陽星座**分 12 種年度建議。  
- **方案 B（完整）**：接入當年重要星象（木土星、日月食、年度過境），或太陽回歸簡化邏輯，再生成「這一程」與工作／感情／自己三句。

第一階段可用**太陽星座 × 當前年**的靜態矩陣（12 × 1 或 12 × 3 句）即可。

## 七、實施優先級建議

| 優先級 | 內容 | 邏輯 | 備註 |
|--------|------|------|------|
| **P0** | 靈魂原型整塊（名稱、關鍵詞、Tagline、金句） | 出生**月、日** → 太陽星座 → 12 套預設文案 | 替換現有五行邏輯，星圖板塊完全西方化 |
| **P1** | 今日宇宙提示 | 月相或「星期 + 太陽星座」→ 預設句 | 替換現有「僅星期」輪播 |
| **P2** | 這一程 + 工作/感情/自己 | 「太陽星座 × 年」或固定年度主題 | 替換現有完全靜態句 |
| **P3** | 月亮星座、上升星座 | 出生時間（+ 出生地）→ 星曆/星盤庫 | 用於更細的個人化與未來合盤 |

## 八、技術實現要點

1. **太陽星座**  
   - 前端或後端用**月、日**依上表計算 `sunSign`（可封裝成 `getSunSign(month, day)`）。  
   - 無需星曆庫，無需調用外部 API。

2. **文案結構**  
   - 建立 12 星座的 i18n 結構，例如：  
     `oracle.archetype.leo.name`、`oracle.archetype.leo.tagline`、`oracle.archetype.leo.quote`、`oracle.archetype.leo.keywords`。  
   - 靈魂原型區塊改為讀取 `oracle.archetype[sunSign].*`，不再讀五行相關 key。

3. **今日 / 年度**  
   - 今日：用當前日期算月相或月亮星座（可接輕量星曆庫），或先用「星期 + sunSign」矩陣。  
   - 年度：用當前年 + sunSign 取預設句，或固定幾條年度主題。

4. **與現有「出生日期」表單的關係**  
   - 現有表單已有年、月、日；L1 不需改表單。  
   - 之後若做 L2/L3，再在表單增加「出生時間」與「出生地」欄位。

---

## 九、小結：我們應該怎麼做、依據什麼邏輯

- **我們應該做**：  
  - 星圖板塊的**所有「靈魂原型」相關內容**，改為依**西方太陽星座**生成（依據出生月、日）。  
  - **今日宇宙提示**改為依**月相或當日星象**（或太陽星座 × 今日）生成。  
  - **這一程**改為依**年度星象或「太陽星座 × 年」**生成。

- **依據的邏輯**：  
  - **數據**：至少出生日期（月、日）；進階再要出生時間與出生地。  
  - **計算**：太陽星座（月日 → 12 星座）；進階再算月亮、上升、月相、過境。  
  - **內容**：預設文案表（12 星座 × 各類文案），由計算結果索引，**不再用八字五行**。

按此方案實施後，星圖板塊的內容生成將與「西方占卜」及西方主流 App 的標準一致，邏輯清晰、可擴展。
